# 🗑️ Laji Monitor - 小区垃圾分类积分自动监控

这是一个基于 Python 和 GitHub Actions 的轻量级网页变化监控脚本。专为监控特定小区垃圾分类积分系统的“每日新增积分”和“排名变化”而设计。

它能在云端定时自动执行，在积分发生变动时通过 Server酱（微信）发送实时通知，并自动生成可视化的每日积分获取报表。

## ✨ 核心功能

- **自动化云端监控**：利用 GitHub Actions 的定时任务（Cron），无需购买服务器即可实现 7×24 小时（或指定时间段）的自动化运行。
- **精准拦截打扰**：经过严格的去重逻辑处理，仅当**今日积分增加**或**可用积分变动**时才发送微信通知。纯粹的被动排名变动会在后台静默更新，拒绝消息轰炸。
- **智能休眠机制**：当检测到当日积分已达到上限（默认 4 分及以上）时，脚本会自动打上完成标记，在当天剩余时间内自动跳过检测，节省资源与 API 额度；跨天后（凌晨0点）自动重置恢复。
- **数据隐私保护**：利用 GitHub Secrets 挂载敏感信息，并且推送至公开仓库的对账文件 `last_data.json` 会经过自动打码（掩码 `***`）脱敏处理，防止个人隐私泄露。
- **自动化历史报表**：自动生成并维护 [`history.md`](./history.md) 文件，直观展示每天的积分达标情况和检测时间。

## 🚀 部署与使用指南

本项目已高度解耦，你可以直接 Fork 本仓库或下载代码进行部署。

### 1. 准备环境变量 (Secrets)
由于项目是公开仓库，请**不要**将姓名和小区 ID 写死在代码中。进入你的 GitHub 仓库的 **Settings -> Secrets and variables -> Actions**，添加以下三个变量：

| Secret 名称 | 说明 | 示例 |
| :--- | :--- | :--- |
| `SERVER_KEY` | Server酱（Turbo版）的 SendKey，用于微信推送 | `SCT123456Txxxxxx...` |
| `TARGET_NAME` | 你要查询的真实中文姓名 | `张三` |
| `VILLAGE_ID` | 目标小区的 ID 代码 | `10126` |

### 2. 调整监控时间 (可选)
默认的监控时间段为 **北京时间每天 18:00 到 21:55，每 5 分钟执行一次**。
如果你需要修改运行时间，请编辑 `.github/workflows/run.yml` 文件中的 `cron` 表达式：
```yaml
  schedule:
    # 填写的必须是 UTC 时间（北京时间减去 8 小时）
    # 示例：UTC 10-13 对应北京时间 18:00-21:55
    - cron: '*/5 10-13 * * *'
```
> ⚠️ **注意**：GitHub Actions 官方对 Cron 的最高频率限制为 `5` 分钟，请勿将其设置为 `* * * * *`（1分钟）。

### 3. 运行测试
完成上述配置后，进入仓库的 **Actions** 页面，选择左侧的 **Rank Monitor** 工作流，点击右上角的 **Run workflow** 进行首次手动运行。
运行成功后，你将：
1. 在微信收到一条包含你当前分数和排名的“初始记录”推送。
2. 仓库根目录自动生成 `last_data.json`（已脱敏数据）和 `history.md`。

## ⚠️ 常见问题说明

1. **为什么运行日志显示“未配置 SERVER_KEY”或没收到微信？**
   - 检查 Secrets 填写时是否不小心复制了前后的空格。
   - 检查当天 Server酱 的免费发送额度（5次）是否已耗尽。
2. **长时间不运行会被 GitHub 暂停吗？**
   - 如果仓库连续 60 天没有任何 push 记录，GitHub 会自动禁用定时任务。不过只要你的积分在变动，脚本就会自动 push 更新数据，属于“活跃仓库”。如果不幸被暂停，可在 Actions 页面手动点击 `Enable workflow` 恢复。
